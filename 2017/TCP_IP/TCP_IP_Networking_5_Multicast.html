TCP/IP Networking

IP Multicast

  • Unicast = send to one destination
  • Multicast = send to a group of destinations

dedicated address space

IPv4 : 224.0.0.0/4 (224.0.0.0 to 239.255.255.255)

IPv6 : ff00::/8

Group identification

  • Any Source Multicast (ASM): the group is identified by the multicast address. Any source can send to this group.
  • Source Specific Multicast (SSM): the group is identified by (s, m) where m is a multicast address and s is a (unicast) source address. Only s can send to this group. (232.0.0.0/8 and ff3x::/32 are SSM)

Operation

Destinations need to explicitly join the multicast group with an IGMP: join (s,m) packet for IPv4 or MLD for IPv6.

  • routers build distribution tree via a multicast routing protocol (PIM‐SM) or by other method since the forwarding table entries can't be aggregated
  • packet multiplication is done by routers
  • Routers do exact match for multicast groups
  • Routers also keep state information for every multicast group

There is no ARP for multicast ! IP multicast address is algorithmically mapped to a multicast MAC address (Several multicast addresses may correspond to same MAC address.

  • Last 23 bits of IPv4 multicast address are used in MAC address
  • Last 32 bits of IPv6 multicast address are used in MAC address

Routing

Widespread : PIM (Protocol Independent Multicast) (dense for small networks or sparse mode for TV distribution for example). PIM-SM uses Reverse Path Forwarding : when a router (such as R1) needs to add a receiver, it sends a PIM/JOIN towards the source, using unicast routing. This creates the distribution tree on the fly.

Security

Mitigations : limit multicast rate and number of groups. Use access lists to control who has access to which group.

SSM is safer and can reject unwanted sources.

In Practice

  • Multicast is good for sources

    • One packet sent for destinations
    • Multiplication is done repeatedly, times

  • Multicast suffers from per-flow state in routers

    • Scalability problems (non aggregation)
    • Possible denial of service attacks

  • Multicast is not supported everywhere, but is (with PIM‐SM):

    • At EPFL and other academic networks
    • Internet TV distribution
    • In some corporate / smart grid networks for news, sensor streaming, time synchronization etc…

  • Works only with UDP, TCP does not work with multicast IP

Conclusion

IP multicast came as an after‐thought and uses a different principle than IP unicast (exact match versus longest prefix match) – is not widely deployed

IP multicast addresses cannot be aggregated

IP multicast require the deployment of a solution to compute the multicast trees between routers (with a multicast routing protocol such as PIM‐SM or with a network management application, SDN)
